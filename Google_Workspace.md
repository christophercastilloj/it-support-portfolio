IT Onboarding: Setting Up Google Workspace for New Hires

Objective

To ensure a secure and standardized onboarding process for all new employees using Google Workspace (formerly G Suite).

Process Workflow

Phase 1: Account Creation

Access the Google Admin Console.

Navigate to Users > Add new user.

Use the naming convention: firstname.lastname@company.com.

Assign the user to the correct Organizational Unit (OU) (e.g., Marketing, Engineering).

Phase 2: Security Configuration

Enroll in 2-Step Verification: Mandatory for all users.

Temporary Password: Generate a secure random password and select "Require password change on next sign-in."

Phase 3: Group & Drive Access

Add the user to relevant Google Groups (e.g., all-staff@, dept-team@).

Grant access to specific Shared Drives.

Assign necessary App Licenses (e.g., Google Workspace Business Plus).

Phase 4: Training Material

Provide the user with the "Welcome to the Platform" video model and the internal FAQ link for common tasks like:

Setting up Gmail signatures.

Integrating Google Calendar with Slack/Zoom.

Recovery Steps

If the user cannot log in initially, verify if their IP is being blocked by a localized firewall or if they need to clear their browser's cache/cookies.
